Creating a Model Improved

Introduction

This script demonstrates the construction of a Mizer model using the methodology outlined in the accompanying paper. It uses the tools provided by the [mizerCalibrate] package which facilitates the conversion of data into a mizer-compatible format and model. The example model represents an assemblage of species in the Celtic Sea, incorporating various sources of empirical data from Lauria et al., (2016), parameters estimates from FishBase (Froese and Pauly, 2024) using the rfishbase package (Boettiger, Lang, and Wainwright, 2012). Predation kernels are parameterised using stomach contents data (Murray, [REFERENCE]), and survey data originates from the IBTS and BTS surveys (ICES, 2017).

Note: This script does not include the manual steps needed to disaggregate species groups or to add any missing species that are required to fully align the model with the Fisheries Management Plan. These steps must be completed separately to ensure all target species are properly represented.

2) Data for a Non-Interacting Model

2.1 Species Parameters

In this first step we will be creating a species parameter object.

First we specify which species to include in our model. For this example we have chosen the following species: herring, cod, haddock, whiting, blue whiting, hake, monkfish, horse mackerel, mackerel, plaice, megrim, and sole.

All species parameters will go into a data frame that must have one row for each species and should have a column named ‘species’ that the contains the common name of the species. Additionally we will add a column named ‘Scientific_name’ containing the scientific names of the species. It is important this ‘Scientific_name’ column is included as it will allow us to easily extract species parameters from FishBase later on.

species_map_lauria <- data.frame(
    species = c("Herring", "Cod", "Haddock", "Whiting", "Blue whiting",

In this tutorial we will introduce many new functions included in the MizerEcopath package, the first of which is [fillDefaultsFromFishBase()](add link to help file). This function provides a streamlined way of extracting a number of species parameters from FishBase. We can then specify which of these parameters we would like to remain in our species params data frame called ‘sp’. In this case these species params include w_max, w_mat, a, b, age_mat, and Length.

In addition to these species params from FishBase we will add four additional parameters which will have equivalent values across all species. These parameters include the exponent of consumption (n), the exponent of respiration (p), the exponent of mortality (d), and the conversion efficiency (alpha). The parameter value estimates of n, p and d are a result of metabolic and ecological theory [see the online mizer course for more information]. The parameter value estimate for alpha comes from the theory that 80% of assimilated food energy becomes biomass (Winberg, 1956) and is the default estimate in many ecopath models [for more information see the Ecopath User_Guide].

These four parameters are simply estimates, that may vary greatly from species to species. It is therefore recommended that in future steps when tuning your model you play around with these values within their recommended ranges as they may vary greatly from their estimated values.

Recommended ranges include:

  • n: 0.75 to 0.7

  • p: 0.75 to 0.7

  • d: -0.25 to -0.3

  • alpha: carnivores 0.7 to 0.9, herbivores 0.4 to 0.6, and detrivores 0.2 to 0.3

temp <- fillDefaultsFromFishBase(species_map_lauria)

# Select and rename parameters
sp <- temp %>%
    select(species, w_max, w_mat, a, b, age_mat, K, Winf, Length) %>%
    mutate(
        n = 0.7, # Exponent of consumption (juvenile growth as feeding 
                 # on resource)
        p = 0.7, # Exponent of respiration (metabolism)
        d = -0.3, # Exponent of mortality (in Mizer n - 1)
        alpha = 0.8 # Ecopath default (conversion efficiency)
    )
colnames(sp)<-c("species","w_max","w_mat","a","b", "age_mat","k_vb","w_inf", "Length","n","p","d","alpha")

2.3 Integrating Biomass, Production, and Consumption from Ecopath

Ecopath data is currently not compatible with our species list as for certain species they specify juveniles and adults. As our size spectrum models follow species throughout all the sizes of their life, this is not necessary and therefore we will input both the juvenile and adult data for a species into a single species row within our params object. To do this we will create a dictionary between model species and Ecopath stanzas.

Now we will add the Ecopath params including biomass (B) in tonnes per km², consumption per biomass (Q/B), and production per consumption (P/Q). These are passed to the function [addEcopathParams()] (add link to help page) which computes production per unit of biomass (P/B) internally from these estimates and stores these values in new columns in the species params data frame. The column names include biomass_observed, consumption_observed, production_observed, and ecopath_groups.

Note: This is not the same as matching full interspecific interactions — we’re importing only the overall species-level empirical rates, not species to species rates which are properties which only emerge once diets and predation kernels are specified.

# Use addEcopathParams to validate and add Ecopath parameters to Mizer species.

3) Creating a Non-Interacting Model

3.1 Initial model

In this next step we will take our species params object ‘sp’ and we will make it into a non-interacting model using the newAllometricParams() function. This function will add many new parameters to the species params of each species. You can view the species params table of the model simply by calling species_params(params_object). If you would like to observe specific columns call species_params(params_object)|>select(column1,column2).

# Set up a multi-species size spectrum Mizer model with 200 size classes, 
# and no interspecific interactions.
p <- newAllometricParams(sp, no_w = 200)

Notice how metabolic respiration rate, feeding level, reproduction level, and the interaction resource are set to 0, and R_max and h are set to Inf, this makes the model non interacting.

3.2 Fishing Mortality and Gear

We will now integrate the catch data into the mizer model. To achieve this we need to calculate the total catch (C) per km squared per year. To do this we must first calculate the total fishing mortality (F) by summing catch across all fleets for each species. Once we have the total catch per km squared per year we can use the function addEcopathCatchTotal(). This function uses the catch data frame exported by Ecopath to set up gear parameters.

# Calculate total fishing mortality (F) by summing across all fleets
# for each species
catch$totalF <- rowSums(catch[, -(1:2)], na.rm = TRUE)

# Calculate total catch (C) per km squared per year by multiplying F by
# biomass (B). Alternatively, use C directly if available.
catch <- 
    left_join(ecopath_params, catch, by = c("Group.name" = "Fleet.group")) |> 
        mutate(TotalCatch..t.km..year. = totalF * Biomass..t.km.2.) |> 
        select(Group.name, TotalCatch..t.km..year.)

#Integrate catch data into the Mizer model by calculating catchability
# as the ratio of the observed yield to the observed biomass. Default
# gear parameters are added for an aggregated 'total' gear. 
p <- addEcopathCatchTotal(p, catch)
